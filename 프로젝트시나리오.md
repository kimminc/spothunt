# 위치 기반 이벤트 보물찾기 앱 구현계획서

## 1. 앱 개요

포켓몬고를 참고한 위치 기반 이벤트 앱을 만든다.  
단, 포켓몬 IP를 사용하지 않고, 주최자가 직접 이벤트 방을 만들고 특정 구역 안에 캐릭터, 보물, 미션 아이템을 배치하는 방식이다.

참여자는 방 이름을 검색하고, 주최자에게 공유받은 비밀번호를 입력해 입장한다.  
주최자는 참여자가 1명 이상 입장하면 이벤트를 시작할 수 있다.

이벤트가 시작되면 참여자는 지도에서 자신의 위치를 확인하고, 정해진 위치 반경 안에 들어가 아이템을 획득한다.

---

## 2. 핵심 사용자

### 주최자
- 이벤트 방을 만든다.
- 이벤트 방 이름과 비밀번호를 설정한다.
- 이벤트 방식을 선택한다.
  - 모두획득형
  - 경쟁형
- 이벤트 구역을 설정한다.
- 지도 위에 아이템 위치를 등록한다.
- 참여자가 들어오기를 기다린다.
- 참여자가 1명 이상이면 이벤트를 시작한다.
- 이벤트 종료 후 결과를 확인한다.

### 참여자
- 방 이름을 검색한다.
- 비밀번호를 입력해 입장한다.
- 대기방에서 이벤트 시작을 기다린다.
- 이벤트 시작 후 지도에서 아이템을 찾는다.
- 아이템 반경 안에 들어가면 획득 버튼을 누른다.
- 점수와 획득 목록을 확인한다.

---

## 3. 이벤트 방식

### 3-1. 모두획득형

각 참여자는 각 아이템을 1번씩 획득할 수 있다.

예:
- A 아이템을 철수, 영희, 민수 모두 각각 획득 가능
- 단, 같은 사람이 같은 아이템을 중복 획득할 수는 없음

사용처:
- 아파트 행사
- 어린이 행사
- 가족 보물찾기
- 스탬프 투어

---

### 3-2. 경쟁형

아이템마다 정해진 획득 가능 인원만 가져갈 수 있다.

예:
- A 아이템 maxWinners = 1이면 선착순 1명만 획득
- B 아이템 maxWinners = 3이면 선착순 3명만 획득

판정 기준:
- 클라이언트 시간이 아니라 서버 도착 시간 기준
- 서버에서 위치, 이벤트 상태, 참여자 여부, 남은 수량을 검증한 뒤 성공 처리

## 4. 화면 구성

## 4-1. 공통 화면

### 시작 화면
- 주최자로 시작하기
- 참여자로 입장하기

---

## 4-2. 주최자 화면

### 이벤트 방 만들기 화면
입력 항목:
- 방 이름
- 비밀번호
- 이벤트 방식 선택
  - 모두획득형
  - 경쟁형
- 이벤트 설명
- 최대 참여자 수
- 시작 전 대기 여부

버튼:
- 다음

---

### 이벤트 구역 설정 화면
기능:
- 지도 표시
- 현재 위치 표시
- 이벤트 구역 설정

MVP에서는 복잡한 다각형 구역 대신 중심 좌표 + 반경 방식으로 시작한다.

예:
- 중심 위치: 아파트 중앙
- 이벤트 반경: 300m

저장 데이터:
- centerLatitude
- centerLongitude
- boundaryRadiusMeter

---

### 아이템 등록 화면
기능:
- 지도에서 위치 선택
- 아이템 이름 입력
- 아이템 이미지 선택
- 점수 입력
- 획득 반경 입력, 기본 20m
- 경쟁형일 경우 maxWinners 입력

아이템 예:
- 황금 보물상자
- 고양이 캐릭터
- 용 캐릭터
- 쿠폰 상자
- 미션 카드

---

### 대기방 화면
표시:
- 방 이름
- 참여자 수
- 참여자 목록
- 이벤트 방식
- 시작 버튼

규칙:
- 참여자가 0명이면 시작 버튼 비활성화
- 참여자가 1명 이상이면 시작 버튼 활성화

버튼:
- 이벤트 시작
- 방 삭제
- 아이템 수정

---

### 진행 현황 화면
표시:
- 현재 참여자 수
- 총 아이템 수
- 획득된 아이템 수
- 참여자별 점수
- 종료 버튼

---

## 4-3. 참여자 화면

### 방 검색 화면
기능:
- 방 이름 검색
- 검색 결과 목록 표시

검색 결과:
- 방 이름
- 이벤트 방식
- 상태
  - 대기중
  - 진행중
  - 종료됨

참여자는 대기중인 방에만 입장 가능하다.

---

### 비밀번호 입력 화면
기능:
- 비밀번호 입력
- 입장 버튼

검증:
- 비밀번호가 맞으면 대기방 입장
- 틀리면 오류 메시지 표시

---

### 참여자 대기방 화면
표시:
- 방 이름
- 주최자 이름
- 이벤트 방식
- 현재 참여자 수
- “주최자가 이벤트를 시작할 때까지 기다려주세요” 문구

이벤트 상태가 RUNNING으로 바뀌면 자동으로 지도 화면으로 이동한다.

---

### 지도 플레이 화면
기능:
- 지도 표시
- 내 현재 위치 표시
- 이벤트 구역 표시
- 아이템 위치 표시
- 내 위치와 가까운 아이템 확인

아이템 표시 규칙:
- 아직 획득하지 않은 아이템 표시
- 획득한 아이템은 흐리게 표시하거나 숨김
- 경쟁형에서 이미 소진된 아이템은 “획득 완료” 상태로 표시

---

### 아이템 획득 화면 또는 팝업
조건:
- 사용자가 아이템 반경 안에 들어오면 획득 버튼 활성화
- 기본 반경은 20m

표시:
- 아이템 이름
- 점수
- 남은 수량
- 획득 버튼

---

### 점수판 화면
표시:
- 내 점수
- 내 획득 목록
- 전체 랭킹
- 참여자별 점수

---

## 5. 이벤트 상태값

EventRoom의 status는 다음 값을 가진다.

- WAITING: 대기중
- RUNNING: 진행중
- ENDED: 종료됨
- CANCELLED: 취소됨

상태 변경 규칙:
- 방 생성 시 WAITING
- 주최자가 시작하면 RUNNING
- 주최자가 종료하면 ENDED
- 주최자가 취소하면 CANCELLED
